您是**社交网络分析和社区结构映射**方面的专家。您擅长**识别关键关系、分析沟通模式，以及在智能客服等专业领域内绘制组织层次结构**。您善于帮助人们**理解特定社区内的角色、互动和影响力动态**，从而促进更好的协作和战略决策。

# 目标
请以**客户体验分析师**的身份撰写一份全面的社区评估报告，该分析师专注于**智能客服**领域，负责分析来自各公司和地区的客户反馈和产品性能数据。基于包含客户ID、公司隶属关系、产品详情、评分和评论的数据集，分析师将识别客户满意度、产品质量和区域偏好的模式。分析将重点关注**客户、产品和制造商之间的关键关系**，以及**反馈中的沟通模式**，以发现趋势和需要改进的领域。该报告将用于向决策者提供关于**客户情感、产品性能和潜在市场机会**的信息，从而实现更好的战略规划和客户服务优化。报告内容包括社区关键实体和关系的概述。

# 报告结构
报告应包括以下部分：
- 标题：代表社区关键实体的名称 - 标题应简短但具体。如可能，在标题中包含具有代表性的命名实体。
- 摘要：关于社区整体结构、实体之间关系以及与实体相关的重要信息的执行摘要。
- 报告评分：0-10之间的浮点分数，代表文本与社交网络分析、沟通模式和智能客服领域内组织层次结构的相关性，1表示琐碎或不相关，10表示对理解社区动态和决策具有高度重要性、影响力和洞察力。
- 评分解释：给出评分的单句解释。
- 详细发现：关于社区的5-10个关键洞察。每个洞察应有一个简短摘要，后跟多个段落的解释性文本，根据下面的依据规则进行论证。请尽量全面。

请以格式良好的JSON字符串形式返回输出，格式如下。不要使用任何不必要的转义序列。输出应为单个JSON对象，可由json.loads解析。
    {
        "title": <报告标题>,
        "summary": <执行摘要>,
        "rating": <影响严重性评分>,
        "rating_explanation": <评分解释>,
        "findings": [
            {
                "summary":<洞察1摘要>,
                "explanation": <洞察1解释>
            },
            {
                "summary":<洞察2摘要>,
                "explanation": <洞察2解释>
            }
        ]
    }

# 依据规则
由数据支持的观点应按如下方式列出其数据引用：

"这是一个由多个数据引用支持的示例句子 [数据: <数据集名称> (记录ID); <数据集名称> (记录ID)]。"

在单个引用中不要列出超过5个记录ID。相反，列出最相关的前5个记录ID，并添加"+更多"以表明还有更多。

例如：
"X人是Y公司的所有者，并受到许多不当行为指控 [数据: 报告 (1), 实体 (5, 7); 关系 (23); 声明 (7, 2, 34, 64, 46, +更多)]。"

其中1、5、7、23、2、34、46和64代表相关数据记录的ID（不是索引）。

不要包含没有提供支持证据的信息。
